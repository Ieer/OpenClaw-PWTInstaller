# Reports 提交策略

`panopticon/reports/` 目录用于保存评估与证据输出。默认情况下，这些文件是运行时产物，不建议直接提交到仓库。

## 默认规则

- 根目录下 `*.json` 默认忽略。
- 仅允许提交：
  - 本文档 `README.md`
  - 规则文件 `.gitignore`
  - `samples/` 下的精选、脱敏样例 `*.json`

## 何时允许提交样例

仅在以下场景提交样例：

- 需要演示评估格式或字段语义
- 需要为 CI/回归提供稳定基线
- 需要在文档中引用固定报告样本

## 样例要求

- 必须脱敏：移除或替换主机、绝对路径、会话内容、潜在个人信息。
- 必须稳定：避免包含高频变化字段（例如运行时随机 ID、时间戳）或将其归一化。
- 必须精简：优先保留最小可验证字段，控制文件体积。

## 推荐流程

1. 在本地生成原始报告（不提交）。
2. 复制到 `samples/` 并做脱敏/归一化。
3. 只提交 `samples/` 下最终样例。

## 注意

如果历史上已跟踪了某些报告文件，新增忽略规则不会自动取消跟踪。可按需执行：

```bash
git rm --cached panopticon/reports/<file>.json
```

再提交即可让该文件受忽略规则控制。
