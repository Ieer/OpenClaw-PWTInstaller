# lessons.md (writing)

## Reusable Lessons

### 内容生产
- 先给结构，再扩写细节
- 重要断言必须可追溯到 sources/artifacts
- 不确定的信息标注"待核实"而非编造

### 技术写作
- 长文档需优先交付提纲（目标读者、结构、关键信息缺口）
- 每次阶段性同步用 4 行：已完成 / 进行中 / 阻塞 / 下一步
- 版本差异说明应明确：改了什么、为什么

### 技术项目
- P0-P3 优先级分级可系统化技术债处理（P0 核心功能突破 → P1 健壮性 → P2 设计系统 → P3 优化）
- 设计系统可提升复用性（PPTX 技能中 4 种设计系统 + 预渲染组件库）
- 资源池化可降低重复开销（浏览器池避免频繁启动成本）
- 测试驱动验证（每个优先级都包含测试验证）
