# Business Model Canvas

## Overview

提供系统化的商业模式分析与设计能力，帮助快速拆解任意企业的商业模式底层逻辑，设计创新商业模式，并进行战略评估与案例拆解。

## Quick Start

**拆解现有商业模式：**
1. 识别9大核心模块（客户细分、价值主张、渠道通路、客户关系、收入来源、核心资源、关键业务、重要合作、成本结构）
2. 分析模块间的依赖关系
3. 定位核心优势与潜在短板

**设计新商业模式：**
1. 明确目标客户与价值主张
2. 构建9大模块画布
3. 评估可行性与盈利逻辑
4. 制作最小可行性商业模型（MBM）

## Core Concepts

### 9大构造块

商业模式画布由9个核心模块组成，理解它们的相互作用是关键：

| 模块 | 说明 | 示例 |
|------|------|------|
| 客户细分 | 为谁创造价值？ | 大众市场、利基市场、多边平台用户 |
| 价值主张 | 提供什么价值？ | 创新、性能、定制化、设计、价格 |
| 渠道通路 | 如何触达客户？ | 直销、店铺、网络、合作伙伴 |
| 客户关系 | 如何与客户互动？ | 个人助理、自助服务、社区、自动化 |
| 收入来源 | 如何盈利？ | 资产销售、使用费、订阅费、租赁费 |
| 核心资源 | 需要什么资源？ | 实体资产、知识产权、人力资源、金融资产 |
| 关键业务 | 需要做什么？ | 生产、解决方案、平台、网络 |
| 重要合作 | 谁是关键伙伴？ | 战略联盟、合作关系、供应商 |
| 成本结构 | 主要成本是什么？ | 固定成本、可变成本、规模经济 |

详细定义和细分维度，见：**[references/core-concepts.md](references/core-concepts.md)**

### 5大经典商业模式式样

1. **非绑定式 (Unbundling)** - 将三类业务分离
2. **长尾式 (Long Tail)** - 服务海量利基市场
3. **多边平台式 (Multi-sided Platform)** - 连接不同客户群体
4. **免费式 (Freemium)** - 免费基础，付费增值
5. **开放式 (Open)** - 开放核心资源共创价值

每种式样的适用场景和实施要点，见：**[references/core-concepts.md](references/core-concepts.md)**

## Eight Core Skills

### SK-01: 画布核心拆解
精准识别9大模块，理清依赖关系，定位核心优势与短板。

### SK-02: 画布实操设计
从零构建或重构商业模式，验证逻辑自洽性与市场可行性。

### SK-03: 式样识别应用
匹配5大经典商业模式式样，为特定场景选择最高效的商业逻辑。

### SK-04: 设计方法落地
通过6大设计方法论，将商业想法转化为具体原型与计划。

### SK-05: 战略分析评估
从盈利性、可持续性、创新性、适配性四个维度评估商业模式。

### SK-06: 全流程操盘
主导从动员到管理的五大阶段，实现商业模式创新项目落地。

### SK-07: 跨场景适配
适配营利组织、非营利组织、个人发展等不同场景。

### SK-08: 案例系统化拆解
拆解标杆案例，提炼可复用的商业规律与避坑指南。

每个技能的详细执行标准和操作指南，见：**[references/eight-skills.md](references/eight-skills.md)**

## Design Methods

### 6大设计方法论流程

1. **客户洞察** - 访谈调研，挖掘潜在需求
2. **创意构思** - 头脑风暴，生成创意方案
3. **可视思考** - 转化为草图或画布初稿
4. **原型制作** - 构建最小可行性商业模型（MBM）
5. **故事讲述** - 将模型转化为商业故事
6. **情景推测** - 预设三种外部变化，制定应对策略

详细方法和工具，见：**[references/design-methods.md](references/design-methods.md)**

### 战略评估框架

| 维度 | 评估指标 | 工具/方法 |
|------|----------|-----------|
| 盈利性 | 毛利率、获客成本、回本周期 | 财务建模 |
| 可持续性 | 客户忠诚度、壁垒高度、复购率 | 波特五力模型 |
| 创新性 | 模式新颖度、技术融合度 | 蓝海战略画布 |
| 适配性 | 团队能力匹配、资源匹配 | SWOT分析 |

详细评估流程，见：**[references/design-methods.md](references/design-methods.md)**

## Execution Principles

执行任何商业模式分析时，必须遵循以下原则：

1. **客户中心原则** - 所有分析与设计以客户价值创造为起点，杜绝"自嗨式"方案
2. **模块化协同原则** - 任一模块的调整，必须同步评估对其他8个模块的连锁影响
3. **可视化优先原则** - 优先使用图表、画布、流程图呈现结果，文字仅作补充
4. **动态迭代原则** - 商业模式无标准答案，需明确标注"假设条件"，基于反馈持续优化
5. **实操性第一原则** - 所有建议必须具体、可落地，避免空泛词汇

- 可行性评估
- MBM原型

**战略评估：**
- 四维度评分表

- 画布还原
- 核心归因
- 经验迁移建议
